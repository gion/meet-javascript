javascript...

  • is not a toy language
  • is small but sophisticated
  • is not java (or a subset of java)

Key ideas

  • load & go delivery
  • loose typing
  • objects as general containers
  • prototypal inheritance
  • lambda
  • linkage through global variables

Key ideas : load & go delivery

  • code doesn't need to be compiled
  • code is executed as it loads
  • code type is plain text

Key ideas : loose typing

  • any variable can hold any type of value
  • no compiler is checking types
  • programs tend to be easier to write
  • the language is NOT untyped

Key ideas : objects as general containers

  • unification of objects and hashtables
  • objects are completely dynamic
    • objects new members can be added any time
    • existing members can be edited any time

Key ideas : prototypal inheritance

  • distinct from classical inheritance
  • objects inherit directly from other objects (there are no classes)

Key ideas : lambda

= the use of functions as first class objects

  • functions can have members (just like objects)
  • functions can be stored in variables
  • functions can be passed as arguments

// assign a function to a variable
var f = function(){
    // ...
};

// add a member to a function
f.key = "value";

// pass a function as an argument to another function
f(function(){
    // ...
});

Key ideas : linkage through global variables

  • no real linkage mechanism
  • variables are stored in one global namespace

Values : numbers

  • a single type of number
    • no (unsigned) int/float/double, just number
    • IEEE-754 (aka "double")
  • not good with common arithmetics : 
    • 0.2 + 0.1 != 0.3;
console.log(0.2 + 0.1); // 0.30000000000000004

Values.numbers: NaN

  • it's the result of a erroneous operation
  • NaN is not equal to anithing, including NaN 
    • console.log(NaN == NaN); // false
  • Use the built in isNaN function
    • console.log(isNaN(10 / 0)); // true
  • although NaN stands for NotANumber 
    typeof NaN == 'number';

Values.numbers: Number()

  • converts a value into a number 
    • 123 == Number('123');
  • it returns NaN if it has a problem 
    • isNaN(Number('string')); // true
  • similar to + the prefix operator 
    • +'123' == Number('123');

Values.numbers: parseInt()

  • converts a value into an integer, but stops at the first non-digit character that it encounters 
    • var value = "85px";
console.log(Number(value)); // NaN	
console.log(parseInt(value)); // 85
  • it converts the value to a number in the specified number-base 
    • parseInt('00101', 2) == 5;
  • for floating point values, use parseFloat 
    • var value = "5.2";
console.log(parseInt(value, 10)); // 5  
console.log(parseFloat(value)); // 5.2

Values : strings

  • no separate character type
  • characters can be accessed as array members (the [] notation)
  • both simple and double quotes can be used
  • concatenation is done with the + operator
  • similar strings are equal (==)
  • String.length holds the number of characters
  • strings are immutable
var string1 = "abcd",
    string2 = 'ab' + "cd";

console.log(string1 == string2); // true

console.log(string2.length); // 4

// rewrite the second character 
string1[1] = 'x';
console.log(string1); // it's not "axcd", it's still "abcd"

Values.string : String()

  • it converts a number value to string
    • var number = 13,
    string = String(number); // "13"

Values.boolean : Boolean()

  • returns true if the value is truthy

Use it with caution!

false != Boolean("false");

Values : null

  • a value that isn't anything

Values : undefined

  • not even null
  • the default value for variables and parameters
  • the value of missing members in objects
  • the returning value of functions that do not return anything
var a;
console.log(a); // undefined

function log(msg){
    console.log(msg);
}
log("blah"); // "blah"
log(); // undefined

var obj = {};
console.log(obj.someKey); // undefined

/*void*/ function foo(){ /* some code */ };
var fooResult = foo();
console.log(fooResult); // undefined

Values : objects

everything else is an object...
  • an object is mainly a collection of key-value pairs, where
    • a key can be any string
    • a value can be any value exept undefined

Values : objects

  • can be created:
    • with the new operator 
      var o = new Object(),
    dateObj = new Date();
    • as an object literal (JSON notation) 
      var emptyObject = {},
    bob = {
        "name" : "Bobby McFerrin",
        "age" : 63
    };
    • with the Object.create method
      var o = Object.create(),
    otherObject = Object.create({
        someKey : 'someValue'
    });

Values : objects

  • members can be accessed with dot notation or subscript notation 
    someObject.someKey = someValue;
console.log(someObject['someKey']); // someValue
  • subscript notation is the most general one (the safest) 
    // !!!
var o = {
    'singleWord' : '', 	
    'multiple-words' : ''
};

// OK
o['multiple-words']; 

// ReferenceError: "words" is not defined
o.multiple-words; // translates to o.multiple - words

Values : objects

  • members can be removed with the delete keyword
  • note that only object members can be removed
    • var o = {
    'singleWord' : '', 	
    'multiple-words' : ''
};

// OK
delete o['multiple-words']; 
delete o.singleWord; 

// NOT OK
delete o;

// OK
delete window.o;

Identifiers

  • starts with a letter or "_" or "$"
  • followed by zero or more letters, digits, "_" or "$"
  • by convention, all variables, members and function names:
    • start with lower case (except for constructors)
    • that start with "_" should be reserved for internal use (are private)
    • that start with "$" should be reserved for machines
  • cannot coincide with a reserved word

Operators

    + - * / %

    == != < > <= >=

    && || !

    &| ^ >> >>> <<

Operators.arithmetic: +

  • addition and concatenation
  • if both operands are numbers
        then
            add them
        else
            convert them both to strings and concatenate them 
    • // addition 
1 + 2 == 3;

// conversion and concatenation
'1' + 2 == '12';

// !!
// addition then conversion and concatenation
1 + 2 + '3' + 4 == '334';

Operators.comparison: == and !=

  • classic equal and not equal
  • these operators can do type coercion
  • it's often better to use the strict equality operators (=== and !==), which do not do type coercion

Operators.comparison: == and !=

Operators.logical: &&

  • the guard operator, aka logical AND
  • does not necessarily return a boolean value
    • // returns a boolean, as expected
true && false == false;

// wait, what?
true && "some value" == 'some value';
  • if the first operand is truthy
        then result is second operand
        else result is first operand
  • it can be used to avoid null references
    • if(someObject){
    return someObject.someKey;
} else {
    return someObject;	
}
// can be written as
return someObject && someObject.someKey;

Operators.logical: ||

  • the default operator, aka logical OR
  • does not necessarily return a boolean value
    • // returns a boolean, as expected
true || false == true;

// wait, what?
false || "some value" == 'some value';
  • if the first operand is truthy
        then result is first operand
        else result is second operand
  • it can be used to fill in default values
    • if(userInput){
    myVar = userInput
} else {
    myVar = defaultValue;	
}
// can be written as
myVar = userInput || defaultValue;

Operators.bitwise: &  |  ^  >>  >>>  << 

  • these operators convert the operands to numbers, then convert them to 32-bit signed integers and turn the result back into a 64-bit floating point
  • don't use them if not necessary
    • // this is faster 
10 * 2; // 20;

// than this
10 << 1; //20

Statements: for

  • iterate through all of the elments of an array: 
    • for(var i = 0; i < array.length; i++){
    currentEl = array[i];
    // ... do smth with currentEl
}

Statements: for .. in

  • iterate through all of the members (including inherited ones) of an object: 
    • for(var key in obj){
    if(obj.hasOwnProperty(key)){
        value = obj[key];
        // ... do smth with value
    }
}

Statements: throw

  • throws errors
  • you can use throw with any type of value
    • // throw a custom error
throw new Error(reason);

// throw a custom object
throw {
    name : 'my custom exception',
    message : 'some custom reason'
};

// throw a string
throw "error message";

Statements: try / catch

  • tries to execute a piece of code and catches any exception is thrown only in the corresponding try block
    • try{
    // ...
} catch(e){
    switch(e.name){
        case 'Error':
            // ...
            break;
        case 'someOtherErrorType':
            // ...
            break;
        default:
            throw e;
    }	
}

Statements: try / catch

  • if you want to be able to catch handle most uncatched errors in all of your scripts, use window.onerror
    • window.onerror = function(e){
    // an error has occured
    // do something about it
    alert("We're sorry, but on error has occured. "
        + "Please shut down your pc!");
}

Statements: var

  • defines variables within a function or within the global scope 
    // define variable "name"
var name;

// define variable "someNumber" with the initial value of 13
var someNumber = 13;

// multiple variable definitions with/withouth initial values
var x, y = 'foo', z;
  • setting a value to an undefined variable (not using var) makes that value global 
    // set value 13 to the undeclared variable "someNumber"
someNumber = 13;

// notice what happens
console.log(window.someNumber); // 13
